Catalog example

This example has been designed to demonstrate the most typical usage of Catalog. More information about Catalog can be found in the documentation.

Two subdirectories of the cgi-bin directory contain the configuration files for this example.
The subdirectory admin contains a configuration suited for the administrator of the catalog.
The subdirectory browse contains a configuration for browsing the catalog, all other operations are forbiden.

The catalog_example database has been created when installing the Catalog package (only if it does not already exists) and contains all the catalogs used in this example.

Control panel
The default control panel, without any customization.

Edit URL catalog
Demonstrates the customization of the catalog editing interface. templates.conf is used to map template file names. Templates used :
  • cedit.html (urlcatalog_cedit.html)
  • csearch.html (urlcatalog_csearch.html)

Browse URL catalog (urlcatalog)
Demonstrates the customization of the catalog browsing interface. This example uses intuitive URLs (the URL shows the current category) but the same customization may be used with URLs containing regular cgi-bin parameters. templates.conf is used to map template file names. Templates used :
  • cbrowse_root.html (urlcatalog_cbrowse_root.html)
  • cbrowse.html (urlcatalog_cbrowse.html)
  • csearch.html (urlcatalog_csearch.html)

Chronoligical history of the URL catalog (urldate)
Demonstrates the customization of chronological browsing. The date field used is the creation time.
  • cdate_default.html
  • cdate_year.html
  • cdate_month.html
  • cdate_day.html

Alphabetical URL catalog (urlalpha)
Demonstrates the customization of alphabetical browsing. The comment field is used for lexicographic ordering.
  • calpha_root.html
  • calpha.html
